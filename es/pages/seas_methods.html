MÉTODOS

Se han utilizado los resultados de este proyecto para complementar la información generado por el grupo de predicción estacional de Área de Evaluación y Modelización del Clima de AEMET. Nuestro objetivo consistía en estudiar si los métodos de regionalización estadística ofrecen valor añadido al aplicarlos a los modelos de predicción estacional del ECMWF (ver la sección de Datos para más información). Se seleccionó el método DeepESD debido a su potencial como metodología de aprendizaje profundo y sus buenos resultados (ver la sección de Regionalización para más información) con la intención de regionalizar los pronósticos retrospectivos, verificarlos mediante el cálculo de métricas determinísticas y probabilísticas y, por último, regionalización también los pronósticos futuros.

Aunque el software pyClim-SDM es exhaustivo y útil para regionalizar modelos climáticos globales, las entradas de los modelos estacional son muy específicas y no pueden correrse de forma adecuada. Por tanto, se aplicó directamente deep4downscaling, la librería de Python que pyClim-SDM utiliza para configurar DeepESD. deep4downscaling permite la ejecución de una regionalización estadística mediante aprendizaje profundo a través de herramientas que preprocesan los datos climáticos, transforman y estandarizan las variables, construyen los modelos de entrenamiento de aprendizaje profundo, evalúan métricas de rendimiento y llevan a cabo un postprocesado y un sesgo de corrección. deep4downscaling se encuentra disponible en el siguiente link.

Para una descripción detallada de DeepESD y el funcionamiento de su metodología, ver la sección de Regionalización. En este caso, el modelo DeepESD fue calculado utilizando predictores obtenidos de ERA5 y el predictando obtenido de CERRA para el periodo 1997-2016. Se utilizó una metodología de "dejar uno fuera" (leave-one-out, LOO), en la que el modelo se entrena excluyendo una observación (en este caso, un año, para romper la autocorrelación temporal y evitar resultados demasiado optimistas) y se utiliza para predecir precisamente la observación excluída (el año en cuestión). Esto evita el sobreajuste, simula cómo el modelo se comportaría con datos "independientes" y permite el cálculo de métricas fidedignas. El modelo LOO se aplicó a cada año del pronóstico retrospectivo (25 miembros, 1997-2016); después, un modelo global se aplicó al pronóstico futuro (51 miembros, 2024 y 2025), ambos para sus inicializaciones de octubre, es decir, para el invierno extendido.

Una vez obtenidos los pronósticos retrospectivos regionalizados, estos fueron verificados mediante el siguiente procedimiento:

  1. Cálculo de anomalías año a año con respecto a la climatología del pronóstico retrospectivo (LOO).

  2. Cálculo de las probabilidades tercílicas año a año.

  3. Cálculo de métricas. Se utilizaron las siguientes métricas. El coeficiente de correlación de rangos de Spearman mide la capacidad del modelo para reproducir variabilidad interanual; también se calcularon otras correlaciones (Pearson, que es la medida de dependencia lineal entre dos variables aleatorias cuantitativas). El área bajo la curva característica operativa del receptor (ROC), o AUC, mide cómo de bien el modelo asigna altas probabilidades cuando el tercil predicho ocurre y evalúa cada categoría individualmente; un AUC mayor que 0.5 indica habilidad predictiva, mientras que un AUC de 1 indica discriminación perfecta. La puntuación de probabilidad clasificada (RPS) mide la calidad del pronóstico para categorías ordenadas (terciles inferior, normal y superior), compara el distribución acumulada predicha con la distribución observada y penaliza tanto los errores de categoría como la probabilidad de asignar errores; un RPS de 0 indica un pronóstico perfecto.

Una vez verificados los pronósticos retrospectivos regionalizados, los modelos de pronóstico futuro para dos años (2024 y 2025) fueron también regionalizados y se calcularon sus anomalías y probabilidades tercílicas.

Estos cálculos se llevaron a cabo para agregaciones de 3 y 5 meses.
